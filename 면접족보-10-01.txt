1. 주석이란 무엇이며, 종류는?
작성코드에 대한 설명을 붙여야 할 때 사용하거나 코드 수정 및 변경 시 원본코드 확인이 필요할 때 사용한다.
주석의 종류는 <행 주석 - //>과 <블럭 주석 - /**/> 두 가지가 있다
 
2.주석은 컴파일시 어떻게 되는가?
컴파일에서 제외된다.

3. 들여쓰기는 왜 해야 하는가?
코드 영역을 구분해서 가독성을 높이기 위해

4. 변수란 무엇인가?
데이터 저장을 위한 메모리 공간을 할당받고 접근하기 위한 도구

5. 변수선언의 의미는?
변수에 이름을 주고 용량을 지정한다.

6. Java의 자료형 작성 (8형제) 에 대하여 설명하시오. 
첫째. byte => 1바이트 크기의 정수타입
   
둘째. short => 2바이트 크기의 정수타입
   
셋째. int => 4바이트 크기의 정수타입
   
넷째. long => 8바이트 크기의 정수타입
   
다섯째. float => 4바이트 크기의 실수타입
   
여섯째. double => 8바이트 크기의 실수타입
   
일곱째. char => 2바이트 크기의 문자타입
   
여덟째. boolean => 1바이트 크기의 논리타입(참과 거짓)
   

7. int 형 범위는?
-2,147,483,648~2,147,483,647

8. int num; 을 메모리 그리고 설명해보시오.
 메모리에 num이라고 이름을 붙인 int형의 4바이트 공간이 할당된다.
 

9. java 언어를 창시한 사람은?
제임스 아서 고슬링

10. JDK란 무엇이며, 어디서 다운로드 받으며, OS별로 버전이 있는 까닭은?
java 개발에 필요한 도구로써 Oracle에서 다운받음. 
os가 각각 다르게 설계 되어있어서 각 os가 이해할 수 있는 언어로 변환하려면
os에 맞는 JDK를 다운 받아야함.


11. JVM 무엇의 약자이며,이란 무엇인가?
Java Virtual Machine => 프로그램을 os에 맞춰서 실행시켜주는 역할

12.아래에 대하여 설명하시오.
write once, run everywhere.
Java가 os에서 직접 실행되는것이 아니라 JVM을 각각의 os별로 제공하기 때문에
하나의 소스코드로 자유롭게 사용가능.
